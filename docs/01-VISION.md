# 项目愿景

## 一句话

**让一个资深工程师拥有一支完整开发团队的产出能力。**

## 问题

资深工程师（5-15 年经验）面临的困境：
- 一个人能力强，但产能有限
- 想法多，执行带宽不够
- AI 编程工具（Copilot、Cursor）提升了编码效率，但项目管理、多任务并行、质量保障仍需大量人工精力
- 在多个 AI 窗口间复制粘贴上下文，效率高但"累死了"

## 解决方案

构建一个 AI 编程团队：

- **Leader（AI 编排系统）**：技术经理 + 产品经理，负责需求理解、方案设计、任务分解、质量审查、全局决策
- **团队成员（OpenClaw 实例）**：有自主思考能力的工程师，接受任务后独立完成，遇到困难主动沟通

你只需要：
1. 告诉 Leader 你想做什么
2. 在关键节点做决策
3. 审查最终产出

## 核心原则

### 1. Leader 是大脑，不是所有人

Leader 负责"想"和"管"，不负责"做"。它用最强的模型（Opus）来保证决策质量，但具体编码、测试、部署交给团队成员。

### 2. 团队成员是工程师，不是工具

每个 OpenClaw 实例都有独立思考能力。它们不是简单地执行指令，而是理解任务目标后自主完成工作。遇到问题会主动向 Leader 或同事沟通。

### 3. 任务粒度决定成败

任务拆得越小越清晰，成员执行的成功率越高，成本越低。推荐粒度：0.3-1 小时/任务。

### 4. 人工始终在环

系统最大化自动执行，但人工可以随时介入任何环节。关键决策、质量审查、异常处理都有人工介入点。

### 5. 成本可控

80% 的任务用经济模型（Sonnet），20% 的关键任务用强模型（Opus）。自托管 OpenClaw 免费，主要成本是 API 调用。

## 目标用户

**你自己**——一个资深工程师，希望大幅提升个人开发效率。

这不是 SaaS 产品，不追求商业化。如果未来验证有效，可以考虑开源或产品化，但当前目标纯粹是：**让自己更高效**。

## 与前身项目的关系

前身项目（ai-orchestration）完成了 Stage 0-3 的文档生成工作流，验证了以下核心理念：
- 8 阶段工作流是有效的项目推进框架
- AI 生成 + 人工审查的阶段门控保证质量
- 任务分解是从设计到执行的关键桥梁

本项目吸收这些经验，但做了根本性调整：
- 不再追求 SaaS 商业化，专注个人效率
- 引入 OpenClaw 作为团队成员，实现真正的多 Agent 协作
- 编排系统升级为 Leader 角色，具备 AI 决策能力
- 自举式开发——系统的第一个项目就是构建自身
